# 自动化资源监控工具

一个用于自动化监控和获取受限资源的Python工具，适用于高并发场景下的资源分配系统。
This is to solve the problem where students can't compete with others when booking a specific type of accommodation in XEC dormitories, such as 4‑person rooms. That's why I made a Python program that automatically sends requests. It was tested in practice last year and proven to work.

## 项目简介

本项目是一个网络自动化工具，主要用于监控资源可用状态并在资源可用时自动发起请求。适用于需要定时查询和快速响应的在线资源分配场景。

**典型应用场景**：
- 有限资源的在线分配
- 高并发场景下的自动化监控
- 需要快速响应的状态查询

## 快速开始

### 1. 环境准备

确保已安装 Python 3.7 或更高版本，并安装必要的依赖：

```bash
pip install requests
```

### 2. 配置认证信息

编辑 `config.txt` 文件，填入以下信息：

```
ACCESS_TOKEN=你的访问令牌
COOKIES=你的会话信息
SECKILL_ID=活动标识符
```

**如何获取这些信息？**

访问目标网站并登录，然后：
1. 按 `F12` 打开浏览器开发者工具
2. 切换到 `Network`（网络）标签
3. 刷新页面，观察网络请求
4. 找到任意一个请求，查看请求头（Request Headers）
5. 复制 `x-access-token` 和 `Cookie` 的值

### 3. 运行脚本

```bash
# 基础版本
python dev.py

# 增强版本（推荐）
python test.py
```

### 4. 监控运行

程序启动后会自动：
- 定时查询资源状态（随机间隔8-12秒）
- 检测资源可用性（status=0表示可用）
- 发现可用资源时自动发起请求
- 记录所有操作到日志文件

## 常见问题

### Q: 提示"访问令牌已过期"怎么办？

**A:** 访问令牌有有效期限制（通常数小时到数天），过期后需要重新获取：
1. 重新登录目标网站
2. 按上述方法获取新的令牌
3. 更新 `config.txt` 文件中的 `ACCESS_TOKEN`

### Q: 程序一直显示"没有可用资源"？

**A:** 这可能是因为：
- 确实没有可用资源（被其他人抢先）
- 配置信息错误导致查询失败
- 网络连接问题

建议检查：
- 查看日志文件了解详细错误信息
- 确认配置信息是否正确
- 测试网络连接是否正常

### Q: 如何提高成功率？

**A:** 几个建议：
- 使用 `test.py` 增强版本
- 保持网络连接稳定
- 在资源开放时段提前运行程序
- 不要过度调整请求频率（可能触发限制）

### Q: 可以调整请求频率吗？

**A:** 可以修改代码中的 `time.sleep()` 参数，但请注意：
- 频率过高可能触发服务端限制或封禁
- 频率过低可能错过资源
- 建议保持默认设置（已优化平衡点）
- 遵守目标网站的使用条款

### Q: 程序会被检测到吗？

**A:** 程序已包含以下特性降低检测风险：
- 随机请求间隔（模拟人工操作）
- 完整的浏览器请求头
- 请求频率控制
- 但仍建议合理使用，避免过度自动化
